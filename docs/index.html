What is a Contract-Driven UI?

A Contract-Driven UI is an architectural pattern which allows the UI to focus on the building parameterized components, enabling your application to be constructed in JSON.

The UI implements components and event handlers based on the theme of your application. A contract, or configuration object in the form of JSON, is supplied to specify how your application should be constructed and the parameters associated to those components and event handlers to be built correctly for your user.

Why Clerical?

Clerical is a very simple and basic implementation of a Contract-Driven UI. Clerical abstracts the concept of HTML into JSON, attaching a way to nest configurable components and events with attached state.

Clerical implments routing, state management, and event management to enable you and your team to create the correct web components to allow you to build faster.

Flexibility by Simplicity

Clerical is flexible to support multiple coding styles and patterns. The framework attaches new DOM events to enable easy MV* patterns to be implemented, along with your Web Components to have the flexibility of containing a plethora of parameters.

Clerical was built with as minimal code as possible, utilizing today's ECMAScript standards. That means - when you are building your components and events, you are building them on a layer slightly above raw JavaScript.
